# Writing Rules Checklist

Run this checklist before publishing any document.

## Length
- [ ] Under 750 words (3 min read)
- [ ] If over: cut redundancy first, then compress bullets, then drop weakest section

## Voice
- [ ] Zero em dashes (search for " - " and the unicode character)
- [ ] No glossy phrases ("building momentum," "excited to share," "driving alignment")
- [ ] No LLM filler openers ("Importantly," "Notably," "Interestingly")
- [ ] No hedge stacking ("might potentially be somewhat")
- [ ] No codenames or shorthand that loses meaning without context
- [ ] Every sentence sounds like something a real person would type

## Structure
- [ ] Leads with insight, not evidence inventory
- [ ] Starts with "Why?" not "What?"
- [ ] Trade-offs stated explicitly
- [ ] Actionable next steps clear
- [ ] Bullet points for 3+ related items

## Final read
- [ ] Read it aloud. Does it sound human?
- [ ] Remove any sentence that doesn't carry weight
- [ ] If someone pastes this into an AI and asks "did AI write this?", would the answer be no?
